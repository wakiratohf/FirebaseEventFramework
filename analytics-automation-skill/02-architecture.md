# Architecture — Spec YAML ↔ firebase-events API

## 1. Nguyên tắc cốt lõi

1. **Single source of truth = `event-spec.yaml`.** Tất cả tên screen / button / popup / custom event đều khai báo ở đây. Code generated không bao giờ được sửa tay.
2. **Khớp 1-1 với pattern PROJECT_EVENT_TEMPLATE.md** đã có trong module. Claude Code không invent pattern mới.
3. **Idempotent.** Chạy `/analytics-generate` 2 lần liên tiếp không thay đổi gì.
4. **Validation 2 tầng:**
   - **Spec-time** (Python script): trước khi sinh code, check naming convention + Firebase limits.
   - **Runtime** (Kotlin `EventNameValidator`): nếu lỡ có code sai (member viết tay) → soft warning trong debug build.
5. **Generated files có marker** (`// AUTO-GENERATED by Claude Code skill firebase-events-impl. DO NOT EDIT.`) để audit phát hiện drift.

## 2. Mapping spec YAML → output Kotlin

### 2.1 Khai báo screens

```yaml
screens:
  - id: home                # snake_case, ≤ 40 char
    class: HomeActivity     # Kotlin class hiện có trong project
    package: com.example.vpn.ui.home  # optional, default lấy package gốc
  - id: server_list
    class: ServerListFragment
    parent: home            # optional, dùng cho hierarchical analytics
```

→ Sinh `ScreenName.kt`:
```kotlin
// AUTO-GENERATED. DO NOT EDIT.
package com.example.vpn.event
object ScreenName {
    const val HOME = "home"
    const val SERVER_LIST = "server_list"
}
```

→ Skill cũng emit "hint" cho mỗi screen: "Class `HomeActivity` nên extends `BaseTrackedActivity` và override `screenName() = ScreenName.HOME`." (`/analytics-audit` sẽ tự phát hiện class nào chưa làm.)

### 2.2 Khai báo buttons

```yaml
buttons:
  - screen: home            # foreign-key đến screens[].id
    id: btn_connect         # snake_case
    name: ConnectButton     # human-readable (cho doc/QA)
    popup: rate_dialog      # optional, nếu nút nằm trong popup
    triggers:               # optional, side-effect khi click
      - event: iap_ev
        params:
          where: home_pro_icon
```

→ Bổ sung vào `ButtonName.kt`:
```kotlin
// AUTO-GENERATED. DO NOT EDIT.
object ButtonName {
    const val BTN_CONNECT = "btn_connect"
    // ...
}
```

→ Skill emit "hint" cho member: gắn `AnalyticsEventsUtils.logClickBtn(ScreenName.HOME, ButtonName.BTN_CONNECT)` vào `setOnClickListener` của nút này. Nếu có `triggers`, gắn thêm gọi event tương ứng.

### 2.3 Khai báo popups

```yaml
popups:
  - id: rate_dialog
    class: RateUsDialog        # optional
  - id: permission_request
```

→ `PopupName.kt`:
```kotlin
object PopupName {
    const val NONE = ""
    const val RATE_DIALOG = "rate_dialog"
    const val PERMISSION_REQUEST = "permission_request"
}
```

### 2.4 Khai báo custom event (project-specific)

```yaml
custom_events:
  - name: vpn_connect_ev          # snake_case, ≤ 40 char
    description: "Khi user bấm Connect và nhận kết quả"
    params:
      - name: server_country
        type: string
        sample: "VN, US, JP"
      - name: protocol
        type: string
        enum: [openvpn, wireguard, ikev2]
      - name: connect_result
        type: int
        enum: [0, 1]
        meaning: "0=fail, 1=success"
    trigger_hint:                  # gợi ý cho audit + cho member
      file: VpnConnectionManager.kt
      method: onConnectionResult
```

→ Sinh `project_events/VpnConnectEv.kt`:
```kotlin
// AUTO-GENERATED. DO NOT EDIT.
package com.example.vpn.event.project_events

import android.os.Bundle
import com.tohsoft.firebase_events.models.AnalyticsEvent

/** Khi user bấm Connect và nhận kết quả */
data class VpnConnectEv(
    val serverCountry: String,
    /** openvpn / wireguard / ikev2 */
    val protocol: String,
    /** 0=fail, 1=success */
    val connectResult: Int,
) : AnalyticsEvent {
    override val eventName: String = "vpn_connect_ev"
    override fun toBundle(): Bundle = Bundle().apply {
        putString("server_country", serverCountry)
        putString("protocol", protocol)
        putInt("connect_result", connectResult)
    }

    companion object {
        const val PROTOCOL_OPENVPN = "openvpn"
        const val PROTOCOL_WIREGUARD = "wireguard"
        const val PROTOCOL_IKEV2 = "ikev2"
        const val RESULT_FAIL = 0
        const val RESULT_SUCCESS = 1
    }
}
```

→ Bổ sung helper vào `AnalyticsEventsUtils.kt`:
```kotlin
fun logVpnConnect(
    serverCountry: String,
    protocol: String,
    connectResult: Int,
) {
    AnalyticsEvents.logEvent(
        VpnConnectEv(serverCountry, protocol, connectResult)
    )
}
```

### 2.5 Khai báo user properties bổ sung

```yaml
user_properties:
  - name: preferred_protocol
    type: string
    enum: [openvpn, wireguard, ikev2]
    trigger: "Khi user chọn protocol mặc định trong Settings"
```

→ Bổ sung wrapper vào `AnalyticsEventsUtils.kt` — vì module `firebase-events` đã có sẵn `logUserPropertyEv` private trong `AnalyticsUserProperties`, skill chỉ sinh helper public:

```kotlin
fun setPreferredProtocol(protocol: String) {
    // Khi user chọn protocol mặc định trong Settings
    AnalyticsUserProperties.logCustomUserProperty(
        userPropertyName = "preferred_protocol",
        property = protocol
    )
}
```

*(Nếu `AnalyticsUserProperties` chưa expose helper `logCustomUserProperty`, skill sẽ note thành 1 TODO trong audit report đề nghị mở rộng module — không tự ý sửa source.)*

## 3. Skill xử lý "13 event chuẩn" thế nào?

Skill **không** sinh code cho 13 event built-in (`screen_view_ev`, `click_btn_ev`, `iap_ev`…) vì chúng đã có sẵn trong module. Thay vào đó, skill:

1. Đọc `screens:` → biết có những màn nào → gen `ScreenName.kt` + gợi ý `BaseTrackedActivity`.
2. Đọc `buttons:` → biết có những nút nào → gen `ButtonName.kt` + gợi ý gọi `logClickBtn`.
3. Đọc các flag `has_iap`, `has_ads`, `has_rate_dialog` ở project level → sinh wrapper helper trong `AnalyticsEventsUtils.kt` cho từng nhóm.

```yaml
project:
  name: "VPN Lite"
  package: "com.example.vpnlite"
  has_iap: true              # → gen logIapAttempt() helper
  has_ads: true              # → gen logAdLoad/Show/Click helper
  has_rate_dialog: true      # → gen logRateDialog helper
  has_onboarding: true       # → gen logOnboardingStep helper
  has_appsflyer: false       # → skip AppsFlyer wrapper
```

→ `AnalyticsEventsUtils.kt` chỉ chứa helper cho phần app **thực sự dùng**, không bloat.

## 4. Idempotent generation

Mỗi file sinh ra mở đầu bằng comment header:
```kotlin
// AUTO-GENERATED by skill firebase-events-impl from event-spec.yaml.
// DO NOT EDIT — chạy `/analytics-generate` để cập nhật.
// Hash spec: a1b2c3d4...
```

`Hash spec` = sha256 của YAML normalize. Khi chạy lại `/analytics-generate`:
- Nếu hash trùng + content trùng → no-op
- Nếu hash khác → diff + xin confirm trước khi ghi
- Nếu content khác hash (member sửa tay) → **WARN** drift và đề xuất regen

## 5. Audit phát hiện gì?

Script `audit_project.py` quét Kotlin files trong `app/src/main/`:

| Pattern phát hiện | Action |
|---|---|
| `class XxxActivity : AppCompatActivity` (không phải `BaseTrackedActivity`) | Suggest đổi parent class, kèm `screenName() = ScreenName.XXX` |
| `class XxxFragment : Fragment` không có `logScreenStart`/`logScreenStop` | Suggest gắn vào `onResume`/`onPause` |
| `setOnClickListener { ... }` mà không có `logClickBtn` lân cận (±5 line) | Suggest gắn `logClickBtn` |
| `Dialog.show()` / `BottomSheet.show()` chưa log popup name | Suggest gọi `logScreenStart(..., popupName=PopupName.XXX)` |
| Spec có `custom_event` mà code chưa có `logEvent(XxxEv(...))` nào | Suggest tạo TODO comment ở `trigger_hint.file` |
| Class file Activity/Fragment chưa có trong spec | Suggest user bổ sung vào YAML |

Output: `docs/analytics/audit-report.md` + `.audit-fixes.json` để `/analytics-generate --apply-missing` đọc.

## 6. Verify cho QC

Script `verify_events.py`:
1. Đọc `event-spec.yaml` → sinh test checklist:
   ```markdown
   ## Màn hình Home (HomeActivity)
   - [ ] Mở app → log `screen_view_ev` với screen_name="Home"
   - [ ] Bấm "ConnectButton" → log `click_btn_ev` với name="Home_ConnectButton"
   - [ ] Bấm "PremiumButton" → log `iap_ev` với where="home_pro_icon"
   ```
2. (Tuỳ chọn) Đọc Logcat trong khi tester click → tick auto vào item nào fire đúng.

## 8. Constraint Claude phải tuân thủ

- ❌ KHÔNG sửa file trong `firebase-events/` (module SDK, đã đóng theo SemVer)
- ❌ KHÔNG hard-code string event/screen/button vào Activity/Fragment — phải dùng `ScreenName.X`, `ButtonName.X`
- ❌ KHÔNG bypass `AnalyticsEventsUtils` để gọi thẳng `AnalyticsEvents.logXxx`
- ❌ KHÔNG sửa file có header `AUTO-GENERATED` (sẽ overwrite ở lần generate kế tiếp)
- ✅ KHI custom event chưa có trong module → implement `AnalyticsEvent` interface, không fork module
- ✅ KHI thấy class permission khác app → định nghĩa enum riêng trong `:app`, không sửa `AllowPermission` của module
